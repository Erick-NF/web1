¿Que es la metodologia Incremental?

En esta metodología el cliente puede cambiar los requerimientos del proyecto en cada iteración así mismo, este último puede usar el producto mucho antes de su entrega definitiva


Caracteristicas

Los incrementos son pequeños Permite una fácil administración de las tareas en cada iteración La inversión se materializa a corto plazo Es un modelo propicio a cambios o modificaciones Se adapta a las necesidades que surjan

Ventajas

A medida que los componentes se van incorporando, el cliente tiene la oportunidad de cambiar los requisitos Es un modelo fácil de probar y depurar Se reduce el riesgo de cambios en los requisitos

Desventajas

No es recomendable para casos de sistemas de tiempo real, de alto nivel de seguridad Procesamiento distribuido y de alto índice de riesgos Requiere de mucha planificación Requiere de metas claras para conocer el estado del proyecto
